# Vibe Match — Figma Plugin

Turn a feeling into a design system. Type a vibe phrase like *"Tokyo midnight"* or *"grandma's kitchen"* and Vibe Match calls the Anthropic Claude API to generate a matching color palette, font pairing, and tone words — then applies them directly to your Figma file as local styles.

---

## Getting an Anthropic API Key

1. Go to [console.anthropic.com](https://console.anthropic.com) and sign in (or create a free account).
2. Navigate to **API Keys** in the left sidebar.
3. Click **Create Key**, give it a name, and copy the key — it starts with `sk-ant-`.
4. Keep this key private. It is billed per use.

> **Security note:** Vibe Match stores your API key only in your browser's `localStorage` inside the Figma plugin iframe. It is never sent anywhere except directly to `api.anthropic.com`.

---

## Installing the Plugin Locally in Figma

Figma calls this "loading an unpublished plugin" (previously "load unplugged").

1. **Clone or download** this repository to your machine.
2. **Compile the TypeScript** (see next section) so `code.js` exists next to `manifest.json`.
3. Open **Figma Desktop** (the web version does not support local plugins).
4. Go to the menu: **Figma → Plugins → Development → Import plugin from manifest…**
5. Select the `manifest.json` file inside this folder.
6. The plugin now appears under **Plugins → Development → Vibe Match**.

---

## Compiling TypeScript

You need [Node.js](https://nodejs.org) (v18+) installed.

```bash
# 1. Install dependencies (only needed once)
npm install

# 2. Compile once
npm run build

# 3. Or watch for changes while developing
npm run watch
```

This compiles `code.ts` → `code.js` in the same directory. Figma loads `code.js` as the plugin backend.

> If you only edit `ui.html`, you do **not** need to recompile — just reload the plugin inside Figma.

## How It Works

```
[ui.html]  →  fetch() → api.anthropic.com  →  JSON payload
                                                    ↓
[ui.html]  →  postMessage("apply-styles") → [code.ts]
                                                    ↓
                                          figma.createPaintStyle()
                                          figma.createTextStyle()
```

1. The user types a vibe phrase in the plugin UI.
2. The UI calls the Claude API directly from the browser (`claude-haiku-4-5-20251001`).
3. Claude returns a structured JSON design token payload.
4. The UI validates the JSON, renders a preview, then on "Apply to Figma" sends it to the plugin backend via `postMessage`.
5. The backend creates (or updates) Figma local styles named `VibeMatch/primary`, `VibeMatch/secondary`, etc.

---

## Figma Styles Created

| Style type | Name pattern          | Example             |
|------------|-----------------------|---------------------|
| Paint      | `VibeMatch/{role}`    | `VibeMatch/primary` |
| Text       | `VibeMatch/Heading`   | heading font style  |
| Text       | `VibeMatch/Body`      | body font style     |

Running the plugin again with a new vibe **updates** existing `VibeMatch/` styles rather than creating duplicates. Styles not created by this plugin are never modified.

## Development Tips

- Use **Plugins → Development → Open Console** in Figma Desktop to debug `code.ts` errors.
- The plugin UI is a standard browser iframe — open browser DevTools by right-clicking inside the plugin panel.
- To reset saved history, run `figma.clientStorage.deleteAsync('vibeHistory')` in the Figma console.
- The plugin requests network access only to `https://api.anthropic.com` (declared in `manifest.json`).
